Day-1: On the first day, we started by understanding the insurance dataset in detail. We explored the structure of the data, checked for missing values and duplicate entries, and carefully studied the target variable and features. We went through the statistical summary to get a clear idea of how the data is distributed. This helped us identify the important attributes and plan the preprocessing steps for the project.
Day-2:On the second day, we worked on cleaning the data. We removed duplicate records and handled missing values wherever required. We used the IQR method to detect and manage outliers. We also performed label encoding and necessary feature transformations. By the end of the day, We had a clean and well-prepared dataset ready for analysis and documented all the steps we followed
Day-3:On the third day, we focused on exploratory data analysis. We created a correlation matrix and heatmap to understand relationships between variables. We analyzed how factors like age, BMI, and smoking status affect insurance charges. This helped us gain meaningful insights and select the most important features for model building.
Day-4:On the fourth day, we moved on to building machine learning models. We split the dataset into training and testing sets and implemented Linear Regression, Decision Tree, and Random Forest models. We evaluated their performance using metrics like MAE, RMSE, and R² score. After comparing the results, We selected the best-performing model.
Day-5:On the final day, we finalized and validated the selected model. We compared actual and predicted values to check its performance visually and interpreted the results. We summarized the overall findings, completed the documentation, and prepared the final report for submission
